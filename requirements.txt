# IMPORTANT: pin transformers to 4.47.x exactly.
#   • <4.47 — missing WhisperSdpaAttention; inference takes 600-700s on CPU.
#   • ==4.47.x — WhisperSdpaAttention ships; inference drops to ~50s on CPU.
#   •  >=5.0 — ASR pipeline's preprocess() drops the `num_frames` key that
#             WhisperFeatureExtractor used to return, causing a KeyError crash.
# Do not upgrade transformers without testing a full mic-record round-trip first.
transformers>=4.47.0,<5.0.0

# ── Do NOT install: optimum[openvino] ─────────────────────────────────────────
# Tried as a faster CPU backend, but optimum-intel's OVModelForSpeechSeq2Seq
# can't produce word-level timestamps (confirmed upstream bug,
# github.com/huggingface/optimum-intel issue #561) — generate() doesn't
# return cross_attentions through the OpenVINO decoder, which transformers'
# return_timestamps="word" post-processing requires. Every feature in this
# app (detection, calibration, profiling) depends on word timestamps, so
# this backend is not usable here regardless of its speed. ASR_BACKEND
# defaults to "transformers"; setting ASR_BACKEND=openvino raises a clear
# error rather than attempting transcription. See profiling/asr.py
# _transcribe_openvino's docstring for the full trace.
#

# fine-tune has a different layout. See profiling/asr.py _load_faster_whisper
# for the exact error and reasoning.
